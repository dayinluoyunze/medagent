# 医药Agent平台设计文档

## 1. 项目概述

为医药企业销售代表和用药人员提供智能化服务，主要功能包括产品知识问答、用药指导、销售支持。

**目标用户：**
- 医药代表（销售支持）
- 药店/医院用药人员（用药咨询）

## 2. 功能设计

### 2.1 产品知识问答

回答关于药品的各种问题：

- **基本信息**：适应症、用法用量、禁忌症、注意事项
- **药理药效**：作用机制、药代动力学
- **不良反应**：常见不良反应、特殊人群用药
- **竞品对比**：与同类药品的区别和优势
- **医保准入**：医保目录、招标情况

### 2.2 用药指导（面向用药人员）

- 药品适应症确认
- 用法用量查询
- 注意事项提醒
- 药物相互作用提示
- 特殊人群（孕妇、儿童、肝肾功能不全）用药建议

### 2.3 销售支持（面向医药代表）

- 客户（医生/药店）信息查询
- 拜访准备资料生成
- 产品介绍话术
- 竞品分析
- 销售数据查询（按权限）

## 3. 对话示例

**场景1：用药咨询**
```
用户：这个药饭前还是饭后吃？
Agent：根据说明书，XX药品建议饭后半小时服用，以减少胃肠道刺激。
```

**场景2：销售代表查询**
```
用户：帮我查一下XX医院心内科李主任的信息
Agent：李主任，XX医院心内科主任医师，研究方向为冠心病介入治疗...
```

## 5. 知识库内容

| 知识类型 | 示例 |
|---------|------|
| 药品说明书 | 适应症、用法用量、禁忌 |
| 产品DA/PPT | 推广资料 |
| 竞品资料 | 对比分析 |
| 指南共识 | 临床用药指导 |
| 客户数据 | 医生/药店信息 |

## 6. 安全与合规

- 严格按药品说明书回答，超出范围提示咨询医学团队
- 禁止给出诊断建议
- 禁止承诺疗效
- 数据访问按权限控制
- 完整操作日志

## 7. 部署方式

采用**API调用**方式，简洁易实现，便于快速上线和面试演示。

### 技术选型

- **大模型**：支持多平台API（OpenAI / ModelScope / MiniMax）
- **框架**：LangChain + Streamlit
- **知识库**：本地JSON/Markdown文件 + FAISS向量检索
- **部署**：Streamlit Cloud / Vercel（免费）

### 多平台API适配

```python
# config.py - 统一接口适配

LLM_PROVIDERS = {
    "openai": {
        "api_base": "https://api.openai.com/v1",
        "model": "gpt-4o"
    },
    "modelscope": {
        "api_base": "https://api.modelscope.cn/v1",
        "model": "qwen-plus"
    },
    "minimax": {
        "api_base": "https://api.minimax.chat/v1",
        "model": "abab6.5s-chat"
    }
}
```

只需修改配置文件的`DEFAULT_PROVIDER`即可切换模型。

### 项目结构

```
medagent/
├── app.py                 # Streamlit主应用
├── config.py              # API配置
├── knowledge/             # 知识库文件
│   ├── products.md        # 产品知识
│   ├── drugs.md           # 用药指南
│   └── qa.md              # FAQ
```

### 本地运行

```bash
pip install -r requirements.txt
export OPENAI_API_KEY="your-key"
streamlit run app.py
```
